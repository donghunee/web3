아래는 **제공한 PRD를 그대로 실행 가능한 제품으로 구현하기 위한 ‘화면 단위 바이브 코딩 프롬프트 세트’**다.
각 프롬프트는 **LLM 기반 코드 생성 툴(Cursor, Claude, v0, Replit 등)**에 그대로 넣어도 동작하도록 **UI + 상태 + AI 호출 역할**까지 명시되어 있다.

---

# 전체 화면 구조 요약 (근거 포함)

* **근거**: PRD의 사용자 시나리오(5번 항목)는 “기준 생성 → 기준 확정 → 화면 분석 → 결과 시각화”의 **순차적 인지 흐름**을 요구함
  → 화면도 동일한 단계 구조가 가장 학습 비용이 낮음 (HCI의 Progressive Disclosure 원칙)

1. 서비스 정보 입력 화면
2. 사용성 평가 기준 생성 & 편집 화면
3. 화면 분석 입력 화면
4. 평가 결과 화면 (차트 + 텍스트)

---

# 1️⃣ 화면 1: 서비스 정보 입력 화면

## 목적

* AI가 **도메인 맥락 기반 평가 기준**을 생성하기 위한 최소 정보 수집

## 바이브 코딩 프롬프트

```
너는 UX 리서치 툴을 만드는 시니어 프론트엔드 개발자다.

다음 요구사항을 만족하는 웹 화면을 만들어라.

[화면 목적]
사용자가 자신의 서비스 맥락을 입력하면,
다음 단계에서 AI가 해당 서비스에 맞는 사용성 평가 기준을 생성할 수 있도록 한다.

[UI 요구사항]
- 입력 필드 2개
  1. 서비스 도메인 (예: 금융, 헬스케어, 교육, 커머스 등)
  2. 서비스 설명 (자유 텍스트, 3~5줄 권장)
- CTA 버튼: "사용성 평가 기준 생성하기"

[UX 요구사항]
- 이 단계에서는 분석이나 평가를 수행하지 않는다.
- 사용자가 입력한 정보는 전역 상태로 저장되어 다음 화면에서 사용된다.

[기술적 요구]
- React 기반
- 상태는 context 또는 간단한 store 사용
- 아직 AI 호출은 하지 말고, 다음 화면으로 이동만 처리
```

# 2️⃣ 화면 2: 사용성 평가 기준 생성 & 편집 화면

## 목적

* **고정 프레임 + 도메인 보정**이라는 PRD 핵심을 실제로 구현
* 사용자가 기준 생성 과정에 개입 → 신뢰도 리스크 완화 (PRD 11번 근거)

## 바이브 코딩 프롬프트

```
너는 UX 평가 AI 툴을 만드는 개발자다.

다음 역할의 AI 프롬프트와 UI를 함께 구현하라.

[AI 역할]
- UX 평가자
- 입력: 서비스 도메인, 서비스 설명
- 출력: 사용성 평가 기준 프레임

[고정 상위 카테고리]
1. 편의성
2. 정확성
3. 의미성
4. 유연성
5. 일관성

[AI 출력 규칙]
- 각 상위 카테고리마다 2~4개의 하위 평가 항목 생성
- 도메인 특성에 따라 항목 명칭과 설명을 조정
- 각 항목에는 한 줄 설명 포함

[UI 요구사항]
- 상위 카테고리별 아코디언 UI
- 각 하위 항목:
  - 체크박스 (활성/비활성)
  - 텍스트 수정 가능
- "이 기준으로 평가 시작" CTA 버튼

[UX 요구사항]
- 사용자는 항목을 삭제하거나 비활성화할 수 있어야 한다.
- 최종 확정된 기준만 다음 단계로 전달한다.

[기술 요구]
- 이 화면에서 AI 호출 1회 발생
- AI 응답은 JSON 구조로 받는다.
```

# 3️⃣ 화면 3: 화면 분석 입력 화면

## 목적

* **“화면 목적 + 핵심 Action” 기반 평가**라는 PRD 차별점 구현
* 일반 UX 체크리스트와의 구조적 차이를 명확히 드러냄

## 바이브 코딩 프롬프트

```
너는 사용성 평가 입력 플로우를 설계하는 UX 엔지니어다.

다음 화면을 구현하라.

[입력 필드]
1. 화면 설명 (무엇을 하는 화면인지)
2. 화면의 목적 (이 화면이 성공했다고 판단되는 기준)
3. 사용자의 핵심 Action (사용자가 반드시 수행해야 하는 행동)

[UX 요구사항]
- 입력 예시 placeholder 제공
- 복잡한 UX 용어 사용 금지
- 비전공자도 작성 가능해야 함

[CTA]
- "이 화면 사용성 평가하기"

[기술 요구]
- 이전 단계에서 확정한 평가 기준을 함께 AI로 전달
- 화면 단위로 반복 분석 가능하도록 구조 설계
```

# 4️⃣ 화면 4: 사용성 평가 결과 화면

## 목적

* 점수 **자체보다 근거와 개선 방향이 핵심**이라는 PRD 의도 반영
* 시각화는 상위 요약, 텍스트는 실행 가이드 역할

## 바이브 코딩 프롬프트

```
너는 UX 평가 결과를 시각화하는 프론트엔드 개발자다.

다음 AI 출력과 UI를 구현하라.

[AI 역할]
- 입력:
  - 화면 설명
  - 화면 목적
  - 핵심 Action
  - 확정된 평가 기준
- 출력:
  - 각 하위 평가 항목별:
    - 1~5점 점수
    - 점수 산출 근거 (1~2문장)
    - 문제 원인
    - 개선 제안 (구체적 행동 수준)

[점수 규칙]
- 점수는 상대 평가가 아닌 해당 화면 단독 기준
- 불확실한 경우 3점 사용

[UI 요구사항]
1. 상위 카테고리 기준 5각형 레이더 차트
2. 하위 항목 리스트 테이블
   - 항목명 / 점수 / 근거 / 개선 제안
3. "이 기준으로 다음 화면 평가" 버튼

[UX 요구사항]
- 점수 옆에 반드시 텍스트 근거가 보여야 한다.
- 점수는 참고 지표임을 화면 하단에 명시
```

# 5️⃣ (선택) AI 프롬프트 템플릿 – 평가 전용

> **근거**: PRD 11번 리스크 대응 중 “프롬프트 품질” 명시 → 코드와 분리된 고정 프롬프트 필요

```
너는 UX 전문가다.
다음 정보를 바탕으로 사용성 평가를 수행하라.

[서비스 맥락]
{서비스 설명}

[평가 기준]
{확정된 평가 항목 리스트}

[분석 대상 화면]
- 화면 설명:
- 화면 목적:
- 핵심 Action:

[출력 형식]
- JSON
- 항목별 점수(1~5)
- 점수 근거
- 문제 원인
- 개선 제안

점수는 과장하지 말고,
명확한 근거가 없으면 3점을 사용하라.
```
